1. Создать папку для дальнейшего её преобразования в репазиторий

2. Открыть данную папку в VS Code

3. Прописать команду git init - сделать из папки репазиторий

4. Создать в программе необходимый файл

5. Внести изменения в данный файл и сохранить

6. Прописать команду git add и название файла

7. Прописать команду git commit -a или -m "*комментарий*" - для фиксации данного изменения в программе

8. Также можно использовать следующие команды:

*git status - для понимая текущих изменений, есть ли несохраненные изменения

*git log - посмотреть журнал всех изменений

*git checkout - переключение между commit и branch

*git diff - разница между настоящим состоянием и последним commit

9. Работа с ветками начинается в основной ветке

10. Для создания новой ветки пишем команду git branch "*и название новой ветки*"

11. Переключение между ветками осуществляется с помощью команды git checkout "*и название ветки*"

12. Посмотреть количество и название веток можно с помощью команды git branch

13. Для соединения новой ветки с основной используем команду git merge

14. Для удаления `git branch -d "*и название ветки*"`

15. Часто бывает, что при создании новой ветки и соедиение её с основной может возникать конфликт 

16. Разрешить данный конфликт возможно 4 способами

17. Принять текущий вариант, принять приходящий вариант, принять оба варианта и сравнить изменения

18. Для работы с GitHub создаем профиль на данном ресурсе

19. git clone "*ссылка*" - для работы с удаленными репазиториями

20. git remote add origin "*ссылка*" - связь локального репазитория и сетевого

21. git push -u origin main - отправка в Интернет

22. git pull - из Интернета обратно

23. Конец
